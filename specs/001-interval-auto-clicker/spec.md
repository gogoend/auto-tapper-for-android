# Feature Specification: 自动连点器（Interval Auto-Clicker）

**Feature Branch**: `001-interval-auto-clicker`

**Created**: 2026-06-27

**Status**: Draft

**Input**: User description: "用户通过指定间隔时长来控制每隔 x 秒在屏幕上某位置 tap 一次，用于相机定时拍摄等场景；提供配置界面与可拖动的全局悬浮控制界面；首启需引导授予必需权限，并尽力引导后台保活；需妥善处理来电、锁屏与后台存活等运行期中断。"

## Clarifications

### Session 2026-06-27

- Q: 因锁屏或来电（停止定时模式）而停止后，事件结束（解锁 / 通话结束）时是否应自动恢复定时？ → A: 不自动恢复，需用户手动重新点击"开始"。
- Q: 在定时点击运行过程中修改配置（间隔 / 按下时长 / 来电动作），应何时生效？ → A: 运行期间不允许修改配置；用户尝试修改时弹窗询问是否停止定时，选"是"则停止并允许修改，选"否"则不修改并继续运行。
- Q: 屏幕旋转或分辨率/显示尺寸变化时，已设定的点击位置（准星）应如何处理？ → A: 本次实现先暂停定时点击并提示用户重新确认位置；横屏/竖屏分别记忆不同点击位置的能力作为后续增强，不在本版本范围内。
- Q: 点击间隔的下限，以及"间隔 ≤ 按下时长"的情况如何约束？ → A: 设定一个最小允许间隔并在 UI 强制限制，阻止用户输入过小的值。
- Q: 用户配置与准星位置是否需要跨应用重启持久化？ → A: 配置项（间隔、按下时长、是否立即点击、来电动作）持久化保存；准星位置不持久化，每次启动重置为默认位置。

## User Scenarios & Testing *(mandatory)*

### User Story 1 - 在指定位置定时自动点击（Priority: P1）

用户希望让手机每隔固定时间自动在屏幕上某个位置点击一次，从而无需手动操作即可完成重复点击任务（例如每隔 5 秒按一次相机快门进行定时连拍）。用户设置好间隔时间，将点击位置对准目标（如相机 App 的快门键），点击"开始"后程序便按节奏自动点击；需要时点击"停止"结束。

**Why this priority**: 这是产品的核心价值——"无人值守的定时点击"。没有它产品没有任何意义；它本身即可作为一个可交付、可演示的最小可用版本（MVP）。

**Independent Test**: 打开应用，设置间隔为 5000ms，将准星对准一个可见的目标（如相机快门），点击开始，观察目标位置是否每 5 秒被准确点击一次；点击停止后不再点击。无需依赖拖动调整、可视化倒计时等功能即可验证。

**Acceptance Scenarios**:

1. **Given** 已设置间隔为 5000ms 且准星已对准目标位置，**When** 用户点击"开始"，**Then** 系统在准星所在位置每隔约 5 秒自动执行一次点击，直到被停止。
2. **Given** "开始后立即执行一次点击"设为"是"，**When** 用户点击"开始"，**Then** 系统立即执行第一次点击，随后按设定间隔继续。
3. **Given** "开始后立即执行一次点击"设为"否"，**When** 用户点击"开始"，**Then** 系统等待一个完整间隔后才执行首次点击。
4. **Given** 定时点击正在运行，**When** 用户点击"停止"，**Then** 系统立即停止后续自动点击，控制按钮恢复为"开始"形态。
5. **Given** 准星覆盖在目标应用（如相机）之上且其中心存在半透明控制按钮，**When** 定时器触发一次点击，**Then** 点击作用于准星下方的目标应用，而非悬浮界面自身。
6. **Given** 定时点击正在运行，**When** 用户尝试修改配置，**Then** 系统弹窗询问是否停止定时；选"是"则停止并进入可编辑状态，选"否"则保持运行且配置不变。

---

### User Story 2 - 实时调整点击位置（Priority: P2）

用户希望在不重启任务的情况下，随时把点击位置移动到屏幕上的任意目标，并且在悬浮界面被拖到屏幕边缘时各控件仍然可见、可操作。

**Why this priority**: 不同应用、不同布局下目标位置各异，灵活定位显著提升可用性；但在已能完成核心定时点击（P1）之后才需要，因此次于 P1。

**Independent Test**: 拖动拖拽手柄把悬浮界面移动到屏幕中央、四边、四角等不同位置，确认准星中心即为点击落点，且拖到边缘时拖拽手柄、退出按钮、开始/停止按钮均完整可见且可点击。

**Acceptance Scenarios**:

1. **Given** 悬浮界面已显示，**When** 用户拖动拖拽手柄，**Then** 整个悬浮界面（含准星）随手指实时移动，准星中心即为新的点击落点。
2. **Given** 悬浮界面被拖至屏幕边缘或角落，**When** 默认排布会导致某些按钮被屏幕边界裁切，**Then** 系统重新排布按钮，使所有控件保持在屏幕可见区域内且可点击。
3. **Given** 定时点击正在运行，**When** 用户拖动准星到新位置，**Then** 后续的自动点击作用于新位置。

---

### User Story 3 - 点击节奏可视化与精细配置（Priority: P3）

用户希望直观看到"距离下次点击还有多久"，并能微调每次点击的按下时长，以适配对触发时长敏感的目标应用。

**Why this priority**: 提升体验与可控性，属于锦上添花；缺少它不影响核心定时点击的完成。

**Independent Test**: 运行定时点击，观察停止按钮的扇形角度随时间从满圆均匀收缩到 0，并在归零时刻发生点击；在配置界面修改"按下时长"，确认点击的按下持续时间随之变化。

**Acceptance Scenarios**:

1. **Given** 间隔设为 4 秒且定时点击运行中，**When** 距下次点击剩余 4 秒，**Then** 停止按钮显示为完整圆（约 360°）；剩余 3 秒时约为 270°；剩余 0 秒时约为 0°，随后触发点击并重新填满。
2. **Given** 用户在配置界面设置了某一"按下时长"，**When** 定时器执行一次点击，**Then** 每次点击的按下持续时间与该设置一致。

---

### User Story 4 - 中断与生命周期处理（来电、锁屏、后台存活）（Priority: P2）

定时拍摄等场景往往持续较长时间，期间可能出现来电或屏幕锁定。用户希望程序能合理应对这些中断：来电时按其设置停止或继续，避免误触电话界面；锁屏时停止点击；同时程序在后台/锁屏期间不被系统杀死，以便用户回来后能继续操作。

**Why this priority**: 长时间无人值守运行是核心使用场景，若来电误点电话界面或进程被系统回收导致任务中断，将严重破坏可靠性；但它建立在核心定时点击（P1）之上，故为 P2。

**Independent Test**: 运行定时点击：模拟来电（默认"停止定时"）验证点击停止且不误点电话界面；将来电动作改为"继续运行"验证仍按间隔点击；运行中锁屏验证定时点击停止；锁屏一段时间后解锁验证程序进程仍存活、界面可继续操作。

**Acceptance Scenarios**:

1. **Given** 来电动作设为"停止定时"，**When** 定时点击运行中发生来电，**Then** 系统停止定时点击。
2. **Given** 来电动作设为"停止定时"且已切换到来电界面、此刻本应触发一次点击，**When** 来电发生，**Then** 该次点击不执行（不得点击到电话界面）。
3. **Given** 来电动作设为"继续运行"，**When** 定时点击运行中发生来电，**Then** 系统继续按设定间隔执行点击。
4. **Given** 定时点击运行中，**When** 屏幕锁屏或息屏，**Then** 系统停止定时点击。
5. **Given** 程序处于后台或锁屏状态，**When** 经过一段时间，**Then** 进程不被系统杀死，解锁后用户可继续操作。

---

### User Story 5 - 首启权限与保活引导（Priority: P1）

由于自动点击依赖显示在其他应用之上的悬浮窗与模拟点击等系统授权，用户首次启动时希望被清晰地引导完成必要授权；同时希望被提醒开启后台运行/省电豁免，以便长时间任务不被系统中断——但即便不开启后台/省电设置，程序仍应可基本使用。

**Why this priority**: 必需权限（如悬浮窗、模拟点击）是核心定时点击（P1）能够运行的前提，缺失则程序完全不可用，因此与 P1 同级。后台/省电引导虽为可选增强，但其引导逻辑在同一首启流程中实现，故合并到本故事。

**Independent Test**: 在未授予必需权限的设备上首次启动，验证程序引导用户前往系统授权入口、且在授予前核心功能（开始点击）不可用；在已授予必需权限但未开启后台运行/省电豁免时启动，验证程序弹出可选提示并提供跳转入口，且关闭提示后程序仍可正常配置与开始点击。

**Acceptance Scenarios**:

1. **Given** 设备尚未授予必需权限（如悬浮窗显示），**When** 用户首次启动程序，**Then** 程序检测到缺失并引导用户前往对应系统设置授权；在授予前，核心功能（开始定时点击）不可用。
2. **Given** 用户在引导后拒绝授予必需权限，**When** 返回程序，**Then** 程序保持"不可用"状态并持续提示需要授权，而非静默失败。
3. **Given** 必需权限已授予，但检测到"允许后台运行"未开启或"省电策略"非"无限制"，**When** 用户启动程序，**Then** 程序弹出可选提示，说明不设置可能导致长时间运行被系统中断，并提供跳转到本程序系统应用设置界面的入口。
4. **Given** 必需权限已授予且用户忽略了后台/省电的可选提示，**When** 用户继续使用，**Then** 程序仍可正常配置并开始定时点击（不因未开启后台/省电设置而阻断）。

---

### Edge Cases

- 当点击间隔接近"按下时长"时，由 UI 强制的最小间隔（≥ 按下时长 + 合理缓冲）保证一次点击可完整完成后再进入下一周期，避免重叠（见 FR-024）。
- 系统或前台应用临时抢占资源，导致某一次点击被延迟或错过时 → 系统**不得**在恢复后补发多次堆积的点击，而应丢弃过期的点击并按下一个周期继续。
- 用户拒绝授予**必需**权限（悬浮窗显示、模拟点击）时 → 程序保持不可用并持续引导授权，而非静默失败。
- 用户忽略/拒绝**可选**的后台运行、省电豁免设置时 → 程序仍可基本运行，但长时间后台/锁屏运行时可能被系统回收。
- 运行过程中必需权限被系统撤销时的行为（应停止点击并重新引导授权）。
- 屏幕旋转或分辨率/显示尺寸变化时 → 系统暂停（停止）定时点击并提示用户重新确认点击位置（见 FR-027）。
- 悬浮界面被拖入状态栏、导航栏、刘海/挖孔等系统保留区域时的可点击性。
- 来电动作设为"继续运行"时，点击会作用于来电界面（由用户选择承担），系统不额外拦截。
- 因来电（停止定时模式）或锁屏而停止后，事件结束（通话结束 / 解锁）时不自动恢复，需用户手动重新开始（见 FR-026）。

## Requirements *(mandatory)*

### Functional Requirements

**配置（Configuration）**

- **FR-001**: 用户必须能够设置自动点击的时间间隔，默认值为 5000ms，且不得低于系统设定的最小间隔（见 FR-024）。
- **FR-002**: 用户必须能够设置每次点击的"按下时长"，默认采用平台标准单击（轻触）的按下时长。
- **FR-003**: 用户必须能够设置"定时器开启后是否立即执行一次点击"，默认值为"是"。
- **FR-016**: 当"立即执行"为"是"时，开始后系统必须立即执行一次点击再进入间隔循环；为"否"时，必须先等待一个完整间隔再首次点击。
- **FR-017**: 用户必须能够设置"定时点击过程中有来电时的动作"，选项为「停止定时」与「继续运行」，默认值为「停止定时」。
- **FR-024**: 系统必须设定一个最小允许间隔（不小于"按下时长"加合理缓冲，以保证单次点击可完整完成后再进入下一周期），并在 UI 上强制限制，阻止用户输入低于该值的间隔。
- **FR-025**: 配置项（间隔、按下时长、是否立即点击、来电动作）必须跨应用重启持久化保存；准星位置不持久化，每次启动重置为默认位置（如屏幕中心）。
- **FR-026**: 定时点击运行期间不允许修改配置；配置项仅在未运行（已停止）状态下可编辑。当用户在运行期间尝试修改配置时，系统必须弹窗询问是否停止定时：选"是"则停止定时并进入可编辑状态，选"否"则保持运行且不修改配置。

**全局悬浮界面（Floating UI）**

- **FR-004**: 系统必须提供一个可显示在其他应用之上的全局悬浮界面。
- **FR-005**: 悬浮界面在任何状态下都必须包含以下元素：点击准星（由中心十字与圆形轮廓组成）、拖拽手柄、退出按钮。
- **FR-006**: 用户必须能够通过拖拽手柄实时移动整个悬浮界面，准星中心即为实际点击落点。
- **FR-007**: 未运行定时点击时，准星中心必须显示圆形"开始"按钮：半透明、覆盖在中心十字之上、半径不超出圆形轮廓。
- **FR-008**: 用户点击"开始"按钮后，系统必须开启定时器并在准星中心位置按设定间隔自动执行点击。
- **FR-009**: 运行定时点击时，准星中心的控制按钮必须呈现为扇形"停止"按钮：扇形角度表示距下次点击的剩余时间（满间隔对应约 360°，随时间均匀收缩，临近点击时趋近 0°，类似时钟扫描），同样半透明、覆盖中心十字且半径不超出圆形轮廓。
- **FR-010**: 用户点击"停止"按钮后，系统必须停止后续自动点击，并将控制按钮恢复为"开始"形态。
- **FR-011**: 用户点击退出（收起）按钮后，系统必须停止定时点击并移除全局悬浮界面（隐藏悬浮窗），但保持运行能力存活，以便用户随后可再次显示悬浮窗。（说明：原"直接退出程序"的语义已调整为"隐藏悬浮窗"——因本程序的点击能力依赖常驻的无障碍服务，彻底退出会导致无法再次显示。）
- **FR-014**: 当悬浮界面被拖至屏幕边缘或角落时，系统必须重新排布拖拽手柄、退出按钮与开始/停止按钮，使所有控件保持在可见区域内且可点击。
- **FR-029**: 配置界面必须提供显示/隐藏悬浮窗的切换入口，按钮文案随当前悬浮窗显隐状态切换（隐藏时显示"显示悬浮窗"，显示时显示"隐藏悬浮窗"）；与悬浮窗上的收起按钮状态保持一致。

**点击行为与调度（Click Behavior & Scheduling）**

- **FR-012**: 自动点击必须作用于准星所在屏幕位置下方的目标应用（如相机快门），而不得误触悬浮界面自身。
- **FR-013**: 调度机制必须避免任务堆积：当某次点击因系统繁忙被延迟时，系统不得在空闲后补偿性地连续触发多个被堆积的点击；过期的点击应被丢弃，并以当前时刻为基准重新计时下一次点击。
- **FR-015**: 由于运行期间不可修改配置（见 FR-026），所有配置更改均在停止状态下完成，并在下一次点击"开始"时生效。

**中断与生命周期（Interruptions & Lifecycle）**

- **FR-018**: 当来电动作设为「停止定时」且发生来电时，系统必须停止定时点击；特别地，若此刻已切换到电话来电界面且本应触发一次点击，则该次点击必须被取消，不得作用于电话界面。
- **FR-019**: 当来电动作设为「继续运行」且发生来电时，系统必须继续按设定间隔执行点击。
- **FR-020**: 当定时点击运行中屏幕锁屏或息屏时，系统必须停止定时点击。
- **FR-027**: 当定时点击运行中检测到屏幕旋转或分辨率/显示尺寸变化时，系统必须暂停（停止）定时点击并提示用户重新确认点击位置。（横屏/竖屏分别记忆点击位置为后续增强，不在本版本范围内。）
- **FR-028**: 因锁屏、息屏或来电（停止定时模式）而停止后，系统不得在事件结束（解锁 / 通话结束）时自动恢复定时点击；恢复须由用户手动重新点击"开始"。
- **FR-021**: 程序必须支持在后台/锁屏期间维持存活并尽最大努力（best-effort）避免被系统回收；其在长时间后台/锁屏下不被杀死的能力依赖用户按 FR-023 的引导开启相应系统设置，但不开启时程序仍可基本运行。

**权限与首启引导（Permissions & Onboarding）**

- **FR-022**: 程序启动时必须检查正常运行所必需的权限（如悬浮窗显示、模拟点击/手势）；若任一必需权限未授予，必须引导用户前往对应系统设置授权，且在授予完成前核心功能（开始定时点击）不可用。
- **FR-023**: 程序启动时应检测"允许后台运行"与"省电策略"；当检测到后台运行未开启或省电策略非"无限制"时，可弹出提示并提供入口引导用户跳转到本程序对应的系统应用设置界面。提示文案必须说明不设置可能导致的后果（如长时间运行时被系统中断）。此设置为**可选项**——用户不设置时程序仍可基本运行，不得因此阻断核心功能。

### Key Entities

- **点击配置（Click Configuration）**: 描述一次定时点击任务的参数，包含点击间隔、每次按下时长、是否开始后立即点击、来电动作（停止定时 / 继续运行）；跨应用重启持久化保存。
- **点击位置（Click Target）**: 准星中心在屏幕上的位置，由用户通过拖动悬浮界面确定，可在运行中改变；不持久化，每次启动重置为默认位置。
- **点击会话（Click Session）**: 描述定时点击的运行状态，包含是否运行中、距下次点击的剩余时间（用于扇形倒计时）、已执行点击次数。
- **权限与系统设置状态（Permission & System Setting Status）**: 描述程序运行所需的授权与系统设置情况，区分必需权限（如悬浮窗、模拟点击，未授予则不可用）与可选设置（后台运行、省电豁免，未设置仅影响长时间存活）。

## Success Criteria *(mandatory)*

### Measurable Outcomes

- **SC-001**: 在设定 5 秒间隔下连续运行 10 分钟，相邻两次点击的实际间隔与设定值的偏差不超过 ±5%。
- **SC-002**: 运行期间不出现因延迟累积导致的"多次点击在同一时刻齐发"现象；倒计时动画与界面保持流畅、无明显卡顿。
- **SC-003**: 新用户从打开应用到完成一次"配置 → 放置准星 → 开始自动点击"的完整流程可在 60 秒内完成。
- **SC-004**: 将悬浮界面拖动到屏幕任意位置（含四角与四边）时，所有控件 100% 完整可见且可点击。
- **SC-005**: 任意时刻扇形所示剩余时间与真实剩余时间的偏差小于该间隔的 5%。
- **SC-006**: 自动点击成功触达目标应用（如成功按下相机快门）的成功率不低于 99%。
- **SC-007**: 在「停止定时」模式下，来电期间发生的点击触达电话界面的次数为 0。
- **SC-008**: 在用户已按引导开启后台运行/省电豁免的前提下，程序在锁屏/后台状态下连续保持存活不少于 30 分钟（期间未被系统回收），解锁后界面可立即恢复操作。
- **SC-009**: 首次启动时若缺少任一必需权限，100% 会引导用户至对应授权入口，且在授予前核心功能不可用；可选的后台/省电提示被忽略时，核心功能仍 100% 可用。

## Assumptions

- 目标平台为 Android 手机。必需权限为"显示在其他应用上层（悬浮窗）"与"模拟点击/手势"，由首启引导用户授予（FR-022），未授予则程序不可用。"允许后台运行"与"省电豁免"为可选增强（FR-023），用于提升长时间存活率，未开启不阻断基本运行。具体的系统设置跳转路径与各 Android 版本/厂商差异在规划阶段处理。
- "按下时长"的默认值采用平台标准单击（轻触）的按下持续时间。
- 一次仅控制一个点击位置（单准星、单目标点）。
- 模拟点击在系统手势层面派发，能够穿透到准星下方的目标应用，而不会被半透明的悬浮控制按钮拦截。
- 应用为本地运行：无需联网、无账户体系、不收集或上传用户数据。
- 最小间隔与按下时长上下限的具体数值在规划阶段确定；最小间隔由 UI 强制（见 FR-024）。
- 横屏/竖屏分别记忆点击位置属于后续增强，不在本版本范围内（见 FR-027）。
