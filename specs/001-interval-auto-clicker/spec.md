# Feature Specification: 自动连点器（Interval Auto-Clicker）

**Feature Branch**: `001-interval-auto-clicker`

**Created**: 2026-06-27

**Status**: Draft

**Input**: User description: "用户通过指定间隔时长来控制每隔 x 秒在屏幕上某位置 tap 一次，用于相机定时拍摄等场景；提供配置界面与可拖动的全局悬浮控制界面。"

## User Scenarios & Testing *(mandatory)*

### User Story 1 - 在指定位置定时自动点击（Priority: P1）

用户希望让手机每隔固定时间自动在屏幕上某个位置点击一次，从而无需手动操作即可完成重复点击任务（例如每隔 5 秒按一次相机快门进行定时连拍）。用户设置好间隔时间，将点击位置对准目标（如相机 App 的快门键），点击"开始"后程序便按节奏自动点击；需要时点击"停止"结束。

**Why this priority**: 这是产品的核心价值——"无人值守的定时点击"。没有它产品没有任何意义；它本身即可作为一个可交付、可演示的最小可用版本（MVP）。

**Independent Test**: 打开应用，设置间隔为 5000ms，将准星对准一个可见的目标（如相机快门），点击开始，观察目标位置是否每 5 秒被准确点击一次；点击停止后不再点击。无需依赖拖动调整、可视化倒计时等功能即可验证。

**Acceptance Scenarios**:

1. **Given** 已设置间隔为 5000ms 且准星已对准目标位置，**When** 用户点击"开始"，**Then** 系统在准星所在位置每隔约 5 秒自动执行一次点击，直到被停止。
2. **Given** "开始后立即执行一次点击"设为"是"，**When** 用户点击"开始"，**Then** 系统立即执行第一次点击，随后按设定间隔继续。
3. **Given** "开始后立即执行一次点击"设为"否"，**When** 用户点击"开始"，**Then** 系统等待一个完整间隔后才执行首次点击。
4. **Given** 定时点击正在运行，**When** 用户点击"停止"，**Then** 系统立即停止后续自动点击，控制按钮恢复为"开始"形态。
5. **Given** 准星覆盖在目标应用（如相机）之上且其中心存在半透明控制按钮，**When** 定时器触发一次点击，**Then** 点击作用于准星下方的目标应用，而非悬浮界面自身。

---

### User Story 2 - 实时调整点击位置（Priority: P2）

用户希望在不重启任务的情况下，随时把点击位置移动到屏幕上的任意目标，并且在悬浮界面被拖到屏幕边缘时各控件仍然可见、可操作。

**Why this priority**: 不同应用、不同布局下目标位置各异，灵活定位显著提升可用性；但在已能完成核心定时点击（P1）之后才需要，因此次于 P1。

**Independent Test**: 拖动拖拽手柄把悬浮界面移动到屏幕中央、四边、四角等不同位置，确认准星中心即为点击落点，且拖到边缘时拖拽手柄、退出按钮、开始/停止按钮均完整可见且可点击。

**Acceptance Scenarios**:

1. **Given** 悬浮界面已显示，**When** 用户拖动拖拽手柄，**Then** 整个悬浮界面（含准星）随手指实时移动，准星中心即为新的点击落点。
2. **Given** 悬浮界面被拖至屏幕边缘或角落，**When** 默认排布会导致某些按钮被屏幕边界裁切，**Then** 系统重新排布按钮，使所有控件保持在屏幕可见区域内且可点击。
3. **Given** 定时点击正在运行，**When** 用户拖动准星到新位置，**Then** 后续的自动点击作用于新位置。

---

### User Story 3 - 点击节奏可视化与精细配置（Priority: P3）

用户希望直观看到"距离下次点击还有多久"，并能微调每次点击的按下时长，以适配对触发时长敏感的目标应用。

**Why this priority**: 提升体验与可控性，属于锦上添花；缺少它不影响核心定时点击的完成。

**Independent Test**: 运行定时点击，观察停止按钮的扇形角度随时间从满圆均匀收缩到 0，并在归零时刻发生点击；在配置界面修改"按下时长"，确认点击的按下持续时间随之变化。

**Acceptance Scenarios**:

1. **Given** 间隔设为 4 秒且定时点击运行中，**When** 距下次点击剩余 4 秒，**Then** 停止按钮显示为完整圆（约 360°）；剩余 3 秒时约为 270°；剩余 0 秒时约为 0°，随后触发点击并重新填满。
2. **Given** 用户在配置界面设置了某一"按下时长"，**When** 定时器执行一次点击，**Then** 每次点击的按下持续时间与该设置一致。

---

### Edge Cases

- 当点击间隔被设为小于或接近"按下时长"时，系统如何取舍（例如保证一次点击完整完成后再计时下一次）？
- 系统或前台应用临时抢占资源，导致某一次点击被延迟或错过时 → 系统**不得**在恢复后补发多次堆积的点击，而应丢弃过期的点击并按下一个周期继续。
- 用户尚未授予所需的系统权限（悬浮窗显示、模拟点击）时，开始按钮应给出明确提示而非静默失败。
- 运行过程中权限被系统撤销时的行为。
- 屏幕旋转或分辨率/显示尺寸变化时，已设定的点击位置如何处理（见待澄清问题）。
- 悬浮界面被拖入状态栏、导航栏、刘海/挖孔等系统保留区域时的可点击性。
- 设备息屏/锁屏期间是否继续点击（见待澄清问题）。

## Requirements *(mandatory)*

### Functional Requirements

**配置（Configuration）**

- **FR-001**: 用户必须能够设置自动点击的时间间隔，默认值为 5000ms。
- **FR-002**: 用户必须能够设置每次点击的"按下时长"，默认采用平台标准单击（轻触）的按下时长。
- **FR-003**: 用户必须能够设置"定时器开启后是否立即执行一次点击"，默认值为"是"。
- **FR-016**: 当"立即执行"为"是"时，开始后系统必须立即执行一次点击再进入间隔循环；为"否"时，必须先等待一个完整间隔再首次点击。

**全局悬浮界面（Floating UI）**

- **FR-004**: 系统必须提供一个可显示在其他应用之上的全局悬浮界面。
- **FR-005**: 悬浮界面在任何状态下都必须包含以下元素：点击准星（由中心十字与圆形轮廓组成）、拖拽手柄、退出按钮。
- **FR-006**: 用户必须能够通过拖拽手柄实时移动整个悬浮界面，准星中心即为实际点击落点。
- **FR-007**: 未运行定时点击时，准星中心必须显示圆形"开始"按钮：半透明、覆盖在中心十字之上、半径不超出圆形轮廓。
- **FR-008**: 用户点击"开始"按钮后，系统必须开启定时器并在准星中心位置按设定间隔自动执行点击。
- **FR-009**: 运行定时点击时，准星中心的控制按钮必须呈现为扇形"停止"按钮：扇形角度表示距下次点击的剩余时间（满间隔对应约 360°，随时间均匀收缩，临近点击时趋近 0°，类似时钟扫描），同样半透明、覆盖中心十字且半径不超出圆形轮廓。
- **FR-010**: 用户点击"停止"按钮后，系统必须停止后续自动点击，并将控制按钮恢复为"开始"形态。
- **FR-011**: 用户点击退出按钮后，系统必须立即退出程序并移除全局悬浮界面。
- **FR-014**: 当悬浮界面被拖至屏幕边缘或角落时，系统必须重新排布拖拽手柄、退出按钮与开始/停止按钮，使所有控件保持在可见区域内且可点击。

**点击行为与调度（Click Behavior & Scheduling）**

- **FR-012**: 自动点击必须作用于准星所在屏幕位置下方的目标应用（如相机快门），而不得误触悬浮界面自身。
- **FR-013**: 调度机制必须避免任务堆积：当某次点击因系统繁忙被延迟时，系统不得在空闲后补偿性地连续触发多个被堆积的点击；过期的点击应被丢弃，并以当前时刻为基准重新计时下一次点击。
- **FR-015**: 配置项（间隔、按下时长、是否立即点击）的更改在下一次点击"开始"时生效。 [NEEDS CLARIFICATION: 运行过程中修改配置是否需要立即生效，还是仅下次开始时生效？]

### Key Entities

- **点击配置（Click Configuration）**: 描述一次定时点击任务的参数，包含点击间隔、每次按下时长、是否开始后立即点击。
- **点击位置（Click Target）**: 准星中心在屏幕上的位置，由用户通过拖动悬浮界面确定，可在运行中改变。
- **点击会话（Click Session）**: 描述定时点击的运行状态，包含是否运行中、距下次点击的剩余时间（用于扇形倒计时）、已执行点击次数。

## Success Criteria *(mandatory)*

### Measurable Outcomes

- **SC-001**: 在设定 5 秒间隔下连续运行 10 分钟，相邻两次点击的实际间隔与设定值的偏差不超过 ±5%。
- **SC-002**: 运行期间不出现因延迟累积导致的"多次点击在同一时刻齐发"现象；倒计时动画与界面保持流畅、无明显卡顿。
- **SC-003**: 新用户从打开应用到完成一次"配置 → 放置准星 → 开始自动点击"的完整流程可在 60 秒内完成。
- **SC-004**: 将悬浮界面拖动到屏幕任意位置（含四角与四边）时，所有控件 100% 完整可见且可点击。
- **SC-005**: 任意时刻扇形所示剩余时间与真实剩余时间的偏差小于该间隔的 5%。
- **SC-006**: 自动点击成功触达目标应用（如成功按下相机快门）的成功率不低于 99%。

## Assumptions

- 目标平台为 Android 手机；应用需要"显示在其他应用上层（悬浮窗）"与"模拟点击/手势"两类系统授权，且假定用户已在系统设置中授予；本规格不覆盖具体的授权引导实现。
- "按下时长"的默认值采用平台标准单击（轻触）的按下持续时间。
- 一次仅控制一个点击位置（单准星、单目标点）。
- 模拟点击在系统手势层面派发，能够穿透到准星下方的目标应用，而不会被半透明的悬浮控制按钮拦截。
- 应用为本地运行：无需联网、无账户体系、不收集或上传用户数据。
- 间隔与按下时长存在合理的上下限（用于防止无意义或危险配置），具体边界值在规划阶段确定。
- 配置更改默认在下一次"开始"时生效（参见 FR-015 的待澄清项）。
