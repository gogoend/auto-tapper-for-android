# Quickstart: 自动连点器

## 环境

- Android Studio（含 AGP 9.2.1 兼容版本）、JDK 11+
- Kotlin 2.2.10、Compose BOM 2026.02.01
- 一台 Android 真机（minSdk 26 / API 26+）；**模拟器对无障碍手势与悬浮窗可用，但相机等目标场景建议真机验证**

## 构建与运行

```bash
# 调试构建并安装到已连接设备
./gradlew :app:installDebug

# 单元测试（纯逻辑：调度器、倒计时、配置校验、布局算法）
./gradlew :app:testDebugUnitTest

# 仪器测试（配置/权限界面）
./gradlew :app:connectedDebugAndroidTest
```

## 首次使用（手动验证路径）

1. 启动 App → 若未启用本应用的无障碍服务，被引导至系统设置开启（必需，否则不可用 — FR-022）。可用判定以服务实际连接为准，支持开关/快捷方式/辅助功能按钮任意方式启用。无需单独的悬浮窗权限（悬浮层用 TYPE_ACCESSIBILITY_OVERLAY）。
2. （可选）若提示后台运行/省电豁免，按需开启；忽略不影响基本使用（FR-023）。
3. 配置：间隔（默认 5000ms，受最小间隔限制）、按下时长、是否立即点击、来电动作（默认"继续运行"）。
4. 显示悬浮层，拖动控制条手柄把准星对准目标（如相机快门）；开始/停止按钮在准星中心。
5. 点准星中心按钮"开始"→ 观察每隔设定时间在目标位置自动点击，扇形随时间收缩。
6. 点"停止"，或触发中断（锁屏 / 来电[停止模式] / 旋转）→ 定时停止，且不自动恢复。

## 验收对照（关键 SC/FR · 手动验收清单 T038）

> 真机逐项核对；可在「诊断与日志」页用"分享/查看"导出日志佐证。

| 验证项 | 对应 | 方法 / 预期 | 状态 |
|--------|------|------------|------|
| 点击真正生效 | FR-012 | 准星压目标，开始后目标被点动（Android 16+ 已修复 obscured 丢弃） | ☐ |
| 落点对齐准星 | R14 | 实际点击点 = 可见准星中心（开 Show taps 核对） | ☐ |
| 间隔精度 ±5% | SC-001 | 设 5s 跑 10 分钟，导出日志比对相邻 CLICK 时间戳（逻辑层已由单测保证） | ☐ |
| 无堆积齐发 | SC-002/FR-013 | 制造卡顿后不出现多次瞬时齐发（单测 ClickScheduler 覆盖） | ☐ |
| 边缘控件可见可点 | SC-004/FR-014 | 拖到四角四边，控制条完整可见可点（单测 placeControlBar 覆盖） | ☐ |
| 来电-停止+隐藏 | FR-018 | 停止模式来电：停止且不点电话界面、悬浮窗隐藏（日志 `INCOMING_CALL ... -> STOP + HIDE`） | ☐ |
| 来电-继续 | FR-019 | 继续模式来电：仍按间隔点击 | ☐ |
| 锁屏停止 | FR-020 | 运行中锁屏 → 停止（日志 `STOP reason=SCREEN_OFF`），不自动恢复 | ☐ |
| 旋转停止+提示 | FR-027 | 运行中旋转 → 停止 + Toast 提示，悬浮窗夹回屏内（日志 `STOP reason=CONFIG_CHANGE`） | ☐ |
| 退出=隐藏可重显 | FR-011 | 悬浮窗 X 收起后，设置页"显示悬浮窗"可再次打开 | ☐ |
| 显示/隐藏切换文案 | FR-029 | 计时中显示"隐藏悬浮窗并停止计时"，未计时显示"隐藏悬浮窗" | ☐ |
| 配置持久化/位置不持久化 | FR-025 | 重启后配置保留、准星回到屏幕中心 | ☐ |
| 无障碍以连接为准 | FR-022/R16 | 用快捷方式启用 → 引导页自动消失、可用 | ☐ |

## 可单测的纯逻辑（无需设备）

- `ClickScheduler`（含无堆积、稳态间隔 ±5%）
- `CountdownModel.remainingFraction`
- `placeControlBar`（控制条边缘 + inset 布局）
