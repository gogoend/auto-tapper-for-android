# Quickstart: 自动连点器

## 环境

- Android Studio（含 AGP 9.2.1 兼容版本）、JDK 11+
- Kotlin 2.2.10、Compose BOM 2026.02.01
- 一台 Android 真机（minSdk 26 / API 26+）；**模拟器对无障碍手势与悬浮窗可用，但相机等目标场景建议真机验证**

## 构建与运行

```bash
# 调试构建并安装到已连接设备
./gradlew :app:installDebug

# 单元测试（纯逻辑：调度器、倒计时、配置校验、布局算法）
./gradlew :app:testDebugUnitTest

# 仪器测试（配置/权限界面）
./gradlew :app:connectedDebugAndroidTest
```

## 首次使用（手动验证路径）

1. 启动 App → 若无悬浮窗权限或未启用无障碍服务，被引导至系统设置开启（必需，否则不可用 — FR-022）。
2. （可选）若提示后台运行/省电豁免，按需开启；忽略不影响基本使用（FR-023）。
3. 配置：间隔（默认 5000ms，受最小间隔限制）、按下时长、是否立即点击、来电动作（默认停止）。
4. 显示悬浮层，拖动拖拽手柄把准星对准目标（如相机快门）。
5. 点"开始"→ 观察每隔设定时间在目标位置自动点击，扇形随时间收缩。
6. 点"停止"或触发中断（锁屏 / 来电[停止模式] / 旋转）→ 定时停止，且不自动恢复。

## 验收对照（关键 SC/FR）

| 验证项 | 对应 | 方法 |
|--------|------|------|
| 间隔精度 ±5% | SC-001 | 设 5s 跑 10 分钟，记录点击时间戳 |
| 无堆积齐发 | SC-002/FR-013 | 制造卡顿后观察不出现多次瞬时齐发（单测 ClickScheduler 覆盖逻辑） |
| 穿透自身按钮 | FR-012 | 准星压在相机快门上，验证快门被按下而非误触按钮 |
| 边缘控件可点 | SC-004/FR-014 | 拖到四角四边，验证所有控件可见可点（单测 arrangeControls） |
| 来电不点电话界面 | SC-007/FR-018 | 停止模式下来电，验证不点到电话界面 |
| 锁屏停止 | FR-020 | 运行中锁屏，验证停止 |
| 配置持久化/位置不持久化 | FR-025 | 重启后配置保留、准星回到中心 |

## 可单测的纯逻辑（无需设备）

- `ClickScheduler`（contracts/click-scheduler.md）
- `CountdownModel.remainingFraction`
- `ClickConfig.validate` / 最小间隔钳制
- `arrangeControls`（边缘布局）
